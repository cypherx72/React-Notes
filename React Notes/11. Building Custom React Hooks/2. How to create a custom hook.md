### Creating a Custom Hook: `useFetch`


### ðŸ§  Why Name It `useFetch`?

- In **React**, a function is treated as a **hook** only if it starts with `use`.
    
- This is enforced by React rules via the linter or bundler config (e.g., ESLint).
    
- Example: Calling `useState` inside an _if-statement_ or _non-hook function_ throws an error â€” even though it's valid JavaScript.
    

```js
function fetch() {
  useState(); // âŒ React will complain
}

function useFetch() {
  useState(); // âœ… React accepts this as a hook
}
```

> âœ… Naming the function `useFetch` ensures React applies **hook rules** to it.

---

### ðŸ” Why Build a Custom Hook?

- To **move logic out** of `App.jsx` or any component.
    
- **Reuse** the logic (like `useEffect`, `useState`, and data-fetching) across multiple components.
    
- **Keep components lean** and focused only on rendering.
    

---

### ðŸ’¡ What Goes Inside the Hook?

- All the logic you previously had inside `useEffect` in `App.jsx`.
    
- The `useEffect` and `useState` calls used to fetch and store data.
    

```js
// Inside useFetch.js
import { useState, useEffect } from "react";

function useFetch(fetchFn, dependencies = []) {
  const [fetchedData, setFetchedData] = useState();
  const [isFetching, setIsFetching] = useState(false);
  const [error, setError] = useState();

  useEffect(() => {
    async function fetchData() {
      setIsFetching(true);
      setError(null);
      try {
        const data = await fetchFn();
        setFetchedData(data);
      } catch (err) {
        setError(err.message || "Something went wrong.");
      }
      setIsFetching(false);
    }

    fetchData();
  }, dependencies);

  return {
    fetchedData,
    isFetching,
    error,
  };
}

export default useFetch;
```
