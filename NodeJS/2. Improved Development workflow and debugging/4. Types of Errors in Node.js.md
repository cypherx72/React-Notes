### 2. **Runtime Errors**

> ðŸš« These only show up _after_ your code successfully compiles and tries to run.

- **What causes them?**
    
    - Accessing undefined variables, bad function calls, or failed operations.
        
    - Example:
        
        ```js
        const data = undefined;
        console.log(data.name); // âŒ TypeError: Cannot read property 'name' of undefined
        ```
        
- **How to spot them?**

    - The terminal stack trace is your best friend here.
        
- **Fix?**
    
    - Read stack traces top-to-bottom.
        
    - Use `console.log()` or `debugger` to trace values step-by-step.
